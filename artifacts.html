Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide.

PSV ActivityDefinition (Verification Activity)

Defines a concrete verification activity to be performed by a worker/system.
PSV AuditEvent

System-level audit of verification operations.
PSV Endpoint (US Core derived)

Endpoints for boards/registries/verifiers.
PSV Evidence Binary

Raw binary artifact (hashable).
PSV Evidence Document

Stores document artifacts (PDF, CSV, screenshots) used as evidence.
PSV Measure

Aggregates verification coverage or SLA metrics.
PSV MeasureReport

Reports metrics for verification coverage/SLA.
PSV Organization (US Core derived)

Represents employers, boards (validators), certifying bodies, accreditors.
PSV OrganizationAffiliation

Privileges and affiliations between organizations and practitioners.
PSV PlanDefinition (Policy)

Defines policy (frequency, method, failure action) for verification types.
PSV Practitioner with Qualification Slices (Common Pattern + Rules)

Practitioner profile with typed qualification slices and shared invariants for credentialing.
PSV PractitionerRole (US Core derived)

PractitionerRole for roles/specialties linked to organizations.
PSV Provenance (US Core derived)

Audits VerificationResult with evidence entities (CSV snapshot, API receipt, PDF).
PSV Verification Task

Requests execution of a verification (e.g., LEIE monthly, license quarterly).
PSV VerificationResult

Standard verification evidence linking validators to verified targets.
